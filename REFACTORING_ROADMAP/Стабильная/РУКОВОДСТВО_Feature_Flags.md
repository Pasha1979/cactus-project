# Руководство по использованию Feature Flags (Переключателей функций)

## 📌 Что такое Feature Flags?

**Простыми словами:** Это выключатели для функций приложения. Ты можешь включить или выключить любую функцию без пересборки приложения.

**Зачем нужно:**
- Тестировать новые функции без риска сломать приложение
- Быстро отключать "баганую" функцию (за минуты, а не дни)
- Постепенно внедрять функции для пользователей (сначала 10%, потом всем)

---

## 📋 Какие флаги сейчас доступны?

### ✅ Работающие функции (включены по умолчанию)

| Флаг | Что делает | Где используется |
|------|-----------|------------------|
| `enableGbifParsing` | Поиск информации о растении в GBIF базе | При добавлении/редактировании растения |
| `enableWeatherAdvice` | Показывает советы по поливу на основе погоды | Главный экран, детали растения |
| `enableBatchManagement` | Управление партиями семян/сеянцев | Экран добавления растения |
| `enableQrCodes` | Генерация и сканирование QR-кодов | Меню растения, настройки |
| `enableWinteringNotifications` | Напоминания о начале/окончании зимовки | Система уведомлений |
| `enableAdvancedStatistics` | Расширенные графики и статистика | Экран статистики |

### 🔮 Флаги на будущее (функции ещё не созданы)

| Флаг | Будущая функция | Статус |
|------|----------------|--------|
| `enableNewWateringAlgorithm` | Новый умный алгоритм расчёта полива | ⏳ Не разработана |
| `enableExperimentalPlantCard` | Новый дизайн карточки растения | ⏳ Не разработана |
| `enableAutoCloudSync` | Автоматическая синхронизация с облаком | ⏳ Не разработана |

**Важно:** Флаги на будущее не мешают работе приложения. Они просто ждут, когда функции будут созданы.

## 🚀 Быстрый старт: Как включить/выключить функцию

### Способ 1: Локальное тестирование (для разработчика)

**Где редактировать:** Файл `lib/main.dart` в начале функции `main()`

**Код для включения функции:**

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp();
  await FeatureFlags.initialize();
  
  // 🟢 ВКЛЮЧИТЬ функцию (даже если она выключена по умолчанию)
  FeatureFlags.setLocalOverride(FeatureFlag.enableGbifParsing, true);
  
  // 🔴 ВЫКЛЮЧИТЬ функцию (даже если она включена по умолчанию)
  FeatureFlags.setLocalOverride(FeatureFlag.enableWeatherAdvice, false);
  
  runApp(MyApp());
}
```

**Результат:**
- Работает только на твоём устройстве/эмуляторе
- После перезапуска без изменений кода — вернётся к настройкам по умолчанию
- Идеально для тестирования

---

### Способ 2: Для всех пользователей через Firebase

**Шаг 1: Открыть Firebase Console**
1. Перейти на [https://console.firebase.google.com](https://console.firebase.google.com)
2. Выбрать проект "My Cactus"
3. В левом меню выбрать **"Remote Config"** (или "Удалённая конфигурация")

**Шаг 2: Добавить новый параметр**
1. Нажать кнопку **"Добавить параметр"**
2. В поле "Имя параметра" ввести ключ флага (например: `enable_gbif_parsing`)
3. В поле "Значение по умолчанию" выбрать тип **"Boolean"**
4. Установить значение: `true` (включить) или `false` (выключить)

**Шаг 3: Опубликовать**
1. Нажать **"Опубликовать изменения"**
2. Подтвердить в диалоговом окне

**Шаг 4: Подождать или перезапустить приложение**
- Изменения применяются:
  - На Android: через 1 час или при перезапуске приложения
  - На Windows: через 1 час или при перезапуске
  - Мгновенно: если вызвать `FeatureFlags.forceRefresh()`

## 📝 Примеры использования в разных ситуациях

### Ситуация 1: "Нужно срочно отключить GBIF из-за ошибки"

**Проблема:** GBIF API сломался, и приложение зависает при добавлении растения.

**Быстрое решение (локально):**
```dart
// В main.dart добавить перед runApp:
FeatureFlags.setLocalOverride(FeatureFlag.enableGbifParsing, false);
```

**Решение для всех пользователей:**
1. Зайти в Firebase Console
2. Найти параметр `enable_gbif_parsing`
3. Изменить значение на `false`
4. Опубликовать
5. Приложение перестанет обращаться к GBIF (кэш останется работать)

---

### Ситуация 2: "Хочу протестировать функцию до релиза"

**Задача:** Проверить новый алгоритм полива (когда он будет готов).

**Действия:**
```dart
// В main.dart временно включаем:
FeatureFlags.setLocalOverride(FeatureFlag.enableNewWateringAlgorithm, true);

// После тестирования — удаляем эту строку или меняем на false:
FeatureFlags.setLocalOverride(FeatureFlag.enableNewWateringAlgorithm, false);
```

**Проверка:**
```dart
// В любом месте кода можно проверить статус:
bool isNewAlgorithmOn = FeatureFlag.enableNewWateringAlgorithm.isEnabled;
print('Новый алгоритм: $isNewAlgorithmOn');
```

---

### Ситуация 3: "Альфа-тестирование для избранных"

**Задача:** Дать доступ к новой функции только себе и 2 друзьям.

**Решение через Firebase (с условиями):**
1. В Firebase Console → Remote Config
2. Добавить условие: **"User ID"** contains твой ID
3. Установить значение `true` для этих пользователей
4. Для всех остальных оставить `false`

## 🧪 Код для проверки в приложении

### Проверка в UI:
```dart
// В любом виджете:
if (FeatureFlag.enableGbifParsing.isEnabled) {
  // Показать кнопку поиска в GBIF
  return GbifSearchButton();
} else {
  // Показать сообщение "Функция отключена"
  return DisabledFeatureMessage();
}
```

### Проверка в сервисе:
```dart
// В GbifService (уже сделано):
Future<Map<String, dynamic>?> fetchGbifData(String name) async {
  // Проверяем флаг в начале метода
  if (!FeatureFlags.isEnabled(FeatureFlag.enableGbifParsing)) {
    return null; // Молча возвращаем null, приложение не падает
  }
  
  // ... остальной код
}
```

### Проверка в провайдере:
```dart
// В PlantCrudProvider или другом:
void someMethod() {
  if (FeatureFlag.enableWeatherAdvice.isEnabled) {
    // Использовать новую логику
    _showWeatherAdvice();
  } else {
    // Использовать старую логику
    _showDefaultAdvice();
  }
}
```

## 🔧 Вспомогательные команды

### Принудительное обновление (перезагрузка флагов):
```dart
// Полезно после изменений в Firebase:
await FeatureFlags.forceRefresh();
```

### Сброс всех локальных настроек:
```dart
// Удаляет все override'ы, возвращает к значениям по умолчанию:
FeatureFlags.clearAllLocalOverrides();
```

### Сброс конкретного флага:
```dart
// Удаляет override только для GBIF:
FeatureFlags.clearLocalOverride(FeatureFlag.enableGbifParsing);
```

## 📊 Состояния флагов

### Что видит пользователь:

| Состояние | Для пользователя | Твои действия |
|-----------|------------------|---------------|
| Флаг включён (`true`) | Функция работает | — |
| Флаг выключен (`false`) | Функция скрыта/не работает | — |
| Ошибка загрузки Firebase | Используется значение по умолчанию | Проверить интернет |

### Что видит разработчик:

```dart
// В коде можно узнать источник значения:
bool isEnabled = FeatureFlag.enableGbifParsing.isEnabled;

// Если true — функция включена
// Если false — функция выключена (или Firebase не загрузился)
```

## ❗ Важные правила

### ✅ Правильно:
1. **Всегда проверяй флаг в начале метода** — не в середине
2. **Возвращай null/safe значение при выключенном флаге** — не бросай исключение
3. **Логируй** — пиши в лог когда функция отключена
4. **Тестируй оба состояния** — и включен, и выключен

### ❌ Неправильно:
```dart
// ❌ НЕ ТАК:
void doSomething() {
  var data = fetchData(); // Ошибка если флаг выключен!
  if (FeatureFlag.enableXxx.isEnabled) {
    useData(data);
  }
}

// ✅ ПРАВИЛЬНО:
void doSomething() {
  if (!FeatureFlag.enableXxx.isEnabled) {
    return; // Ранний выход
  }
  var data = fetchData();
  useData(data);
}
```

## 🆘 Частые проблемы и решения

### Проблема 1: Изменения в Firebase не применяются

**Причина:** Firebase кэширует значения на 1 час.

**Решение:**
```dart
// Принудительно обновить:
await FeatureFlags.forceRefresh();
```

### Проблема 2: В debug режиме игнорируется Firebase

**Причина:** В `kDebugMode` используются значения по умолчанию (для тестирования).

**Решение:** Используй `setLocalOverride()` для тестирования в debug.

### Проблема 3: Приложение падает при выключенном флаге

**Причина:** Не проверили флаг перед использованием функции.

**Решение:** Добавить проверку:
```dart
if (!FeatureFlag.xxx.isEnabled) {
  return null; // или showPlaceholder();
}
```

## 📞 Памятка на скорую руку

### Включить функцию для теста:
```dart
FeatureFlags.setLocalOverride(FeatureFlag.ИМЯ_ФЛАГА, true);
```

### Выключить функцию для теста:
```dart
FeatureFlags.setLocalOverride(FeatureFlag.ИМЯ_ФЛАГА, false);
```

### Проверить включена ли функция:
```dart
if (FeatureFlag.ИМЯ_ФЛАГА.isEnabled) {
  // делать что-то
}
```
