# ЧЕК-ЛИСТЫ ДЛЯ РЕФАКТОРИНГА

**Дата создания:** 2026-05-10
**Цель:** Систематическая проверка для предотвращения ошибок

---

## ЧЕК-ЛИСТ: Проверка плана ПЕРЕД изменениями

### Общий анализ
- [ ] Прочитать цели фазы (в 02-REFACTORING_PLAN.md)
- [ ] Прочитать финальную архитектуру (в 01-AFTER_REFACTORING.md)
- [ ] Прочитать LESSONS_LEARNED.md для проверки на похожие ошибки
- [ ] Понять зависимость изменения от других шагов

### Анализ изменения
- [ ] Какова цель изменения?
- [ ] Какие шаги затрагиваются?
- [ ] Какие шаги добавляются/удаляются/перемещаются?
- [ ] Соответствует ли изменение целям фазы?
- [ ] Соответствует ли изменение финальной архитектуре?

### Проверка на риски
- [ ] Что может сломаться если это изменение выполнено?
- [ ] Есть ли зависимости от этого изменения?
- [ ] Нужно ли изменить другие шаги?
- [ ] Есть ли риск потери данных?

### Проверка на логичность
- [ ] Логичен ли порядок шагов после изменения?
- [ ] Не нарушает ли изменение зависимости между шагами?
- [ ] Есть ли более логичный способ достичь цели?

## ЧЕК-ЛИСТ: Проверка плана ПОСЛЕ изменений

### Проверка целостности
- [ ] Проверить все шаги Фазы 1 (grep "^### 1\.")
- [ ] Проверить все шаги Фазы 2 (grep "^### 2\.")
- [ ] Проверить все шаги Фазы 3 (grep "^### 3\.")
- [ ] Проверить отсутствие дубликатов номеров шагов
- [ ] Проверить последовательность нумерации (без пропусков)

### Проверка на случайные удаления
- [ ] Все ли шаги из ДО изменения на месте?
- [ ] Не удален ли критически важный шаг?
- [ ] Не удален ли шаг, который соответствует цели фазы?

### Проверка на соответствие целям
- [ ] Все ли шаги соответствуют целям своей фазы?
- [ ] Нет ли шагов из других фаз в текущей фазе?
- [ ] Есть ли дубликаты между фазами?

### Проверка нумерации задач
- [ ] Нумерация задач внутри шагов соответствует номеру шага?
- [ ] Нет ли задач с неправильной нумерацией (например, 1.10.x в шаге 1.11)?

### Проверка на нелогичности
- [ ] Логичен ли порядок шагов?
- [ ] Есть ли шаги, которые должны быть раньше/позже?
- [ ] Не нарушена ли логика зависимостей?

- [ ] Изменения нужны для Windows?

- [ ] Нет ли неиспользуемых импортов?

- [ ] Все задачи шага выполнены?
- [ ] Работают ли изменения на Android?
- [ ] Работают ли изменения на Windows?
- [ ] Не сломаны ли существующие функции?

### Проверка плана
- [ ] Обновлен ли план с учетом выполненного шага?
- [ ] Отмечен ли шаг как выполненный в 03-CURRENT_REFACTORING_STEP.md?
- [ ] Нужно ли обновить план на основе уроков?

### Обучение
- [ ] Были ли ошибки при выполнении?
- [ ] Записаны ли ошибки в LESSONS_LEARNED.md?
- [ ] Проанализированы ли ошибки глубоко?
- [ ] Добавлены ли новые правила на основе ошибок?

## ЧЕК-ЛИСТ: Обязательный 8-пунктный чек-лист

1. [ ] flutter analyze проходит без ошибок (Android)
2. [ ] flutter analyze проходит без ошибок (Windows)
3. [ ] Все задачи шага выполнены
4. [ ] Код соответствует доменной модели
5. [ ] Код соответствует целям рефакторинга
6. [ ] Нет хардкода, все вынесено в константы
7. [ ] Нет дублирования кода
8. [ ] Код соответствует SOLID принципам

## ЧЕК-ЛИСТ: Глубокий анализ перед решением

- [ ] Какие есть возможные решения?

### Анализ решений

- [ ] Каковы минусы каждого решения?
- [ ] Какое решение наиболее логично?
- [ ] Какое решение соответствует целям рефакторинга?

- [ ] Проверить решение еще раз
- [ ] Есть ли более лучшее решение?
- [ ] Решение единственно верное?

- [ ] Записать уроки
