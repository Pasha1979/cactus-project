### Правило работы с заглушками и TODO

**Дата:** 2026-05-09
**Категория:** Планирование
**Контекст:** Работа над проектом в целом

**Описание правила:**
1. Если ставлю заглушку - обязательно помечать её, чтобы потом видеть где стоят заглушки и что их надо реализовывать
2. В конце каждого шага, задачи, подпункта код должен быть чистым и все TODO в коде по возможности реализованы
3. Если на данном этапе нет возможности реализовать какой-то TODO или его реализация может привести к ошибке, записывать реализацию TODO в план в те моменты когда TODO может быть реализован без ошибок

**Пример правильной заглушки:**
```dart
// TODO(1.2): Реализовать кэширование фото - после завершения задачи 1.1
bool get isCached => false;
```

**Пример неправильной заглушки:**
```dart
// TODO: реализовать флаг синхронизации
bool get isSyncing => false;
```

**Урок:**
- ВСЕГДА помечать заглушки с ссылкой на задачу/шаг
- В конце каждого шага проверять код на наличие TODO
- Если TODO нельзя реализовать сейчас - записать в план когда можно
- НЕ оставлять немаркированные заглушки в коде

**Связанные задачи:**
- Все задачи проекта
- Файл: ROADMAP/LESSONS_LEARNED.md

### Пакет printing требует Visual Studio 2022 build tools

**Дата:** 2026-05-09
**Категория:** Окружение
**Контекст:** Тестирование Windows приложения после добавления printing зависимости

**Описание ошибки:**
- При запуске `flutter run -d windows` ошибка CMake/MSBuild
- `error MSB8020: The build tools for Visual Studio 2022 (Platform Toolset = "v143") cannot be found`
- Установлена Visual Studio 18 Insiders (предварительная версия)
- Пакет `printing` требует компиляции нативного кода с VS2022 build tools v143
- Build tools v143 не установлены или несовместимы с Insiders

**Причина ошибки:**
- Пакет `printing` требует нативной компиляции на Windows
- Нужна Visual Studio 2022 с C++ build tools v143
- Insiders версия может не иметь совместимых build tools

**Исправление:**
- Вариант 1: Установить Visual Studio 2022 (не Insiders) с C++ build tools
- Вариант 2: Временно удалить `printing` из pubspec.yaml (QR-генерация работает, печать нет)
- Вариант 3: Использовать альтернативный пакет без нативной компиляции

**Урок:**
- Проверять требования пакетов к окружению перед добавлением
- Пакеты с нативной компиляцией требуют правильного окружения
- Тестировать сборку на целевой платформе сразу после добавления зависимостей

**Связанные задачи:**
- Задача 1.1: QR-этикетки
- Файл: Windows/pubspec.yaml (printing зависимость)
